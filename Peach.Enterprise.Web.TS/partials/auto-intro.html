Automation

The Pit Automation section allows you to tell Peach to take certain actions during fuzzing iterations that you would not be able to initiate manually, like electronically flip a hardware switch or starting an process that resides on either the local or remote machine.
The Pit Automation section is optional. If you don't have the appropriate hardware or don't want to initiate a process while fuzzing, you may skip this section.
If you want to automate more than one action, you will need to repeat Pit Automation section process for each action.
You'll need the following information to complete the Pit Automation section:

  • Whether you want to automate the task on a local machine or remote machine
  • If the automation is done on a remote machine, the remote machine's hostname/IP name and OS
  • Which process do you want to automate (CanaKit Relay board or process running on the selected machine)
  • Serial port, and related relay information (CanaKit Relay)
  • Executable name and location, arguments and actions to take (process)
