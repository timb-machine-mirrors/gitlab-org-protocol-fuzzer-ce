When a fault is detected, Peach Fuzzer Professional can log additional information to assist in analyzing the fault to determine the root cause. This information can include:

  • Log files stored on disk or on remote machines via SSH
  • Network traffic captures that can be loaded into Wireshark
  • Serial console output from devices such as routers or switches

The Pit Data Collection section is optional. If you don't produce this kind of data or if you choose not to capture it, you may skip this section.

If you want to monitor more than one file, you will need to repeat Pit Data Collection section process for each item you would like to collect.

You'll need the following information to complete the Pit Fault Detection section:

  • If the detection is done on a remote machine, the remote machine's hostname/IP name and OS
  • The kind of data collection (network capture, locally generated file, remote file)
  • Network device implement (network capture)
  • Output file name (locally generated file)
  • User name, authentication type and and required information, and files or folders to be downloaded (remote file)
