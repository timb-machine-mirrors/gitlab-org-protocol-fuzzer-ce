Does your target produce extra files or network information that Peach may want to capture such as:

  • Disk log files?
  • Network traffic?
  • Console output from appropriate hardware?

If you produce data from target files, network traffic, or console output, completing the Pit Data Collection section process for each file or network or output allows Peach to collect data to help you diagnose the root cause of a crash.
The Pit Data Collection section is optional. If you don't produce this kind of data or if you choose not to capture it, you may skip this section.
If you want to monitor more than one file, you will need to repeat Pit Data Collection section process for each file.
You'll need the following information to complete the Pit Fault Detection section:
  • Whether you want to detect a fault on a local or remote machine
  • If the detection is done on a remote machine, the remote machine's hostname/IP name and OS
  • The kind of data collection (network capture, locally generated file, remote file)
  • Network device implement(network capture)
  • Output file name (locally generated file)
  • User name, authentication type and and required information, files or folders to be downloaded (remote file)
